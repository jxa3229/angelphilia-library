## 2. 项目发布结构

公开仓库以 `D:\Project\angelphilia-library` 为根目录，直接发布该目录，不再保留外层 `angelphilia/` 嵌套。

关键文件：

- `package.json`：构建脚本入口
- `scripts/extract-library-data.mjs`：从 `index_local.html` 提取 `src/data/records.json`
- `scripts/extract-docx-details.mjs`：从 `records/*/record.docx` 提取 `public/details.json`
- `scripts/prepare-public.mjs`：复制 `records/` 图片与档案到 `public/media/`
- `.github/workflows/pages.yml`：GitHub Pages 自动部署
- `vite.config.js`：使用 `BASE_PATH=/angelphilia-library/`

## 3. 本地构建命令

在 `D:\Project\angelphilia-library` 下执行：



构建成功后会生成：

- `public/details.json`
- `public/media/`

## 5. GitHub Pages 启用方式

### 5.1 正确设置

GitHub 仓库页面进入：

`Settings -> Pages -> Build and deployment`

`Source` 必须选择：

- `GitHub Actions`

不要保留：

- `Deploy from a branch`

因为本项目使用的是 Actions workflow，而不是直接从分支目录发布。

### 5.2 工作流文件

已落地文件：

- `.github/workflows/pages.yml`

核心逻辑：

1. `actions/checkout@v4`
2. `actions/configure-pages@v5`
3. `actions/setup-node@v4`
4. `npm ci`
5. `npm run build`
6. `actions/upload-pages-artifact@v3`
7. `actions/deploy-pages@v4`

环境变量：

```text
BASE_PATH=/angelphilia-library/
```

## 7. 本次实际排障记录

### 问题 1：Pages workflow 已存在，但仓库仍未启用

现象：

- `actions/configure-pages@v5` 报错
- 报错核心为 `Get Pages site failed`

根因：

- 仓库的 `Settings -> Pages -> Source` 仍是未启用状态
- 没有切到 `GitHub Actions`

解决：

- 手动到 `Settings -> Pages`
- 把 `Source` 从 `Deploy from a branch` 切为 `GitHub Actions`
- 重新运行失败的 workflow

### 问题 2：workflow 首次失败后需要重新运行

处理方式：

在 Actions 失败页面中执行：

- `Re-run jobs`
- `Re-run all jobs`

之后 build 与 deploy 均成功。

### 问题 3：Node.js 20 deprecation warning

当前状态：

- 不影响本次部署成功
- 只是 GitHub Actions 的未来兼容性提醒

后续可做：

- 关注 `actions/checkout`、`actions/configure-pages`、`actions/setup-node`、`actions/deploy-pages` 的新版本
- 后续统一升级到兼容 Node 24 的 action 版本

## 8. 详情页相关注意事项

详情页不是手写静态 JSON，而是从每个档案目录下的 `record.docx` 自动提取：

- 源文件：`records/{folder}/record.docx`
- 构建产物：`public/details.json`
- 运行时读取：前端 `fetch(details.json)`

所以如果详情页为空，优先检查：

1. 该条目是否存在 `record.docx`
2. `npm run sync:details` 是否成功
3. `public/details.json` 是否已生成
4. `dist/details.json` 是否随构建产出
5. 线上路径是否为 `/angelphilia-library/details.json`

## 9. 快速检查清单

每次发布前至少检查：

1. `npm.cmd run build` 本地通过
2. `public/details.json` 已生成
3. `public/media/` 已复制完整
4. `dist/` 已生成
5. GitHub `Pages Source` 仍为 `GitHub Actions`
6. `Actions` 中最新一次 `Deploy to GitHub Pages` 为成功
7. 线上首页能打开
8. 任意一个详情页图片与 docx 正文能正常加载
